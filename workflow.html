VCS, CD, and Workflow

Terminology

  • VCS = Version Control System
  • CD = Continuous Deployment
  • IDE = Integrated Development Environment

Workflow

  1. Change files on laptop using IDE
  2. Add, Commit and Push to VCS (create new version)
  3. Wait for CD robot, then check/test website

Notes

A set of files on a computer aren't a website because they can only be accessed by people on that computer. This is why Hosting is necessary to make it a real website.

When changing software you use an IDE to take away all the tedious tasks and allow you to focus on the actual work. It helps you be more productive, but doesn't do the work for you.

When changing software you keep track of the changes that you make to help track down problems and understand why things have gone wrong.

A version control system keeps track of changes for you and allows you to set up continuous deployment.

The IDE integrates with the VCS to remove a lot of the overhead and make the workflow simple.

Continuous Deployment is where an automated process updates the website for you each time you push a change to the code.

VCS Terminology

  • git is a VCS system (there are many)
  • repository (repo) is a like a folder on a computer
  • add = pick which files to commit
  • commit = make a new entry in the history log
  • push = send all new history log entries to the repo
